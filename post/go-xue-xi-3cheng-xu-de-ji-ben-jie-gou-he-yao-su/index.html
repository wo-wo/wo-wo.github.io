1.程序的基本结构和要素

1.包：package

Go语言使用包来组织源代码，并实现命名空间的管理，每个Go文件属于且仅属于一个包

类似命名空间

1.规范

  • 1.必须在源文件中非注释的第一行指明这个文件属于哪个包
  • 2.包的名称可以跟目录不同名，不过建议同名；
  • 3.同一个目录下的同级的所有go文件应该属于一个包；
  • 4.一个Go语言程序有且只有一个main函数，是入口函数，且必须属于main包；

2.main包限制

  • 一个Go语言程序不能有个多个 package main，否则程序(包)运行(go run 运行包，go run file运行文件)报错main redeclared in this block

2.引入包 import

1.作用域-标识符(常量、变量、类型、函数名、结构字段等等)

1.大写字母开头 => public
2.小写字母开头 => private

2.别名

别名在包名前。例 import fm "fmt"

5.注释

  • 1.行注释 //
  • 2.块注释 /** */
  • 3.package 之前的块注释将被默认认为是这个包的文档说明

6.生成代码文档：godoc

会从 Go 程序和包文件中提取顶级声明的首行注释以及每个对象的相关注释，并生成相关文档

只能获取在 Go 安装目录下 ../go/src 中的注释内容。

# 生成代码文档：
# 注：命令不能终止，否则服务不可访问
godoc -http :6060
  • 查看文档：http://localhost:6060/
  • 当前程序文档：http://localhost:6060/pkg/#thirdparty

7.格式化代码：gofmt

# 格式化源代码
gofmt [-r '<原始内容> -> <替换内容>'] [-w] dir|file

-w      覆盖源代码(没有，只打印格式化结果，不重写)
-r '<原始内容> -> <替换内容>'  内容替换

# 例：将所有文件中 (a) 替换成 a
gofmt -r '(a) -> a' -w *.go

8.其它命令

1.安装包和依赖包：go install

2.升级：go fix

3.单元测试：go test

4.环境变量：go env
