go学习-2.环境搭建

Posted on 2021-10-12，6 min read

[toc]

1.环境变量

1.$GOROOT：GO安装目录

1.默认 $HOME/go。Windows，c:/go，mac，/usr/local/go
注：brew安装后目录：/usr/local/Cellar/go（brew安装不一定是最新的）

2.$GOPATH：工作区

1.默认 = $GOROOT。从 Go 1.1开始，必须修改为其它目录，例：$HOME/go_www
2.工作区需要特定目录：src：项目源文件目录，pkg：使用的包或库，bin：编译好的可执行文件目录

# 配置工作区
# 1.配置环境变量
vi ~/.bash_profile
******
export GOPATH=$HOME/go_www
export PATH=$PATH:$GOPATH/bin
******

source ~/.bash_profile

# 注：电脑重启后，zsh加载的是 ~/.zshrc，而不是 ~/.bash_profile文件，想要环境变量生效，编辑 ~/.zshrc
vi ~/.zshrc
******
source ~/.bash_profile
******

# 3.创建工作区特定目录
# go get会把下载到的资源按 src/pkg/bin 结构正确安装对应目录
mkdir -p $GOPATH/src $GOPATH/pkg $GOPATH/bin

3.查看 go env

# 配置go全局环境变量
# 注：修改bash_profile添加环境变量，并不能生效
# 国内代理（七牛云）
go env -w GOPROXY=https://goproxy.cn,direct
# go模式
go env -w GO111MODULE=auto

2.安装go

1.mac

# 安装
brew install go
# brew安装后目录：/usr/local/Cellar/go。
# 注：编辑器安装后， GOROOT 默认没有 sdk，需要手动勾选 brew 安装的 sdk
# 查看版本
go version

3.简单配置

打开设置：cmd + ,

  • 1.GOROOT：默认：$HOME/go；brew下载后地址：/usr/local/Cellar/go
  • 2.GOPATH：环境变量中配置的GOPATH；默认$HOME/go，v1.1之后必须修改为其它目录
  • 3.go build：打开一个go文件，右上角"add configuration" -> Templates -> go build，勾选对应的配置地址
  • 4.goimports：自动格式化插件。工具 -> File Watchers -> + -> goimports

注：GOPATH，go mod二种模式的配置，参考【go mod】

2.Visual Studio Code

VSCode常见问题汇总

1.安装 - brew
# 安装 VS Code
brew cask install visual-studio-code
# 验证 VS Code 是否安装正确
code .
2.安装拓展

默认 VS Code 只自带了少量功能集成

1.从 github 上手动下载两个仓库

因为有墙的存在，在线安装的话，很多插件会下载失败。
注：代理设置成全局 go env -w GOPROXY=https://goproxy.cn,direct

# 创建目录 $GOPATH/src/golang.org/x/
git clone https://github.com/golang/tools.git
git clone https://github.com/golang/lint.git
2.安装
  • 1.命令安装
# 安装拓展：vscode-go
code --install-extension ms-vscode.go
# 
go get -u github.com/go-delve/delve/cmd/dlv
#
go get -u github.com/golangci/golangci-lint/cmd/golangci-lint
  • 2.插件安装
  • go：go语言插件
  • code runner：编译运行 go 程序
  • delve：调试工具
3.问题

VSCode常见问题汇总（持续更新中）
1.mac 下 vscode 扩展安装目录
~/.vscode/extensions

2.command ‘code-runner.run’ not found

# 修改 codeManager.js，注释掉：micromatch(13行；224-235行)
vi ~/.vscode/extensions/formulahendry.code-runner-0.11.0/out/src/codeManager.js

3.Install All 插件安装 FAILED

使用 vpn 还是失败，使用Go 模块代理

# macOS 或 Linux
echo "export GO111MODULE=on" >> ~/.profile
echo "export GOPROXY=https://goproxy.cn" >> ~/.profile
source ~/.profile

# Windows
$env:GO111MODULE = "on"
$env:GOPROXY = "https://goproxy.cn"
